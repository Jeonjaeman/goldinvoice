# 인보이스 발행 & 매출관리 시스템 — 데모 빌드 브리프 (Claude Code용)

> **이 문서를 첨부한 화이트보드 이미지와 함께 Claude Code에 넣으세요.**
> 이미지는 "고객이 손으로 그린 원본 기획", 이 문서는 "정제된 구현 사양"입니다. 충돌 시 이 문서를 우선합니다.

---

## 0. 한 줄 목표

귀금속(금) 가공·수출 업체용 **인보이스 발행 + 매출·수금 통합 관리 시스템**의 **영업용 데모**를 만든다.
실제 백엔드/DB 없이, **목업 데이터로 전체 업무 흐름과 화면을 클릭해 볼 수 있는 단일 산출물**이면 된다. 이 데모는 클라이언트 영업·요건 확정용 시각 자료다.

## 1. 빌드 지시 (기술/형식)

- **형식: 단일 `index.html` 파일** (가장 우선). Tailwind(CDN) + 바닐라 JS + 목업 데이터를 JS 객체로 내장. 차트는 Chart.js(CDN).
  - 이유: 영업 데모라 의존성 없이 어디서든 열리고 **Vercel에 그대로 드래그 배포** 가능해야 함.
  - 화면 전환은 해시 라우팅 또는 섹션 show/hide로 SPA처럼 동작.
- 대안(원하면): Next.js(App Router) + Tailwind + 목업 데이터. 더 "제품다운" 느낌이 필요할 때만.
- **폰트:** Pretendard (CDN). 전부 한글 UI.
- **반응형:** 데스크톱 우선(ERP 특성), 단 모바일에서 깨지지 않게.
- **데이터:** 전부 in-memory 목업. localStorage 쓰지 말 것(데모 환경 호환). 새로고침 시 초기 목업으로 리셋 OK.
- **버튼 동작:** "엑셀 다운로드 / PDF 다운로드 / 업로드"는 실제 파일 처리 대신 **토스트 알림**("엑셀 파일이 생성되었습니다" 등)으로 처리. 데모임을 자연스럽게.
- **로그인:** 불필요. 바로 대시보드 진입. (원하면 가짜 로그인 화면 1장만)

## 2. 도메인 핵심 — 이걸 알아야 데모 숫자가 진짜처럼 보인다

이 회사는 **금을 가공해 해외로 수출**한다. 그래서 인보이스 금액이 **매일 금 시세에 연동**돼 변동한다.

**핵심 계산식:**

```
금속가(원)   = 24K 금시세(원/g) × 함량비율 × 중량(g)
공임(원)     = 제품·사이즈별 공임 단가 × 중량 또는 건수
인보이스금액 = 금속가 + 공임
수출 표기    = 인보이스금액(원) ÷ 환율  → USD 표기
```

**이격(gap) 관리:** PI(견적) 발행 시점 금시세와 CI(최종 인보이스) 선적 시점 금시세가 다르다 → 그 차액을 정산에 반영. 데모에서 "PI 시세 → CI 시세 → 차액 반영" 흐름을 보여주면 전문성이 살아난다.

**용어:**
- PI = Proforma Invoice(견적/예비 인보이스), CI = Commercial Invoice(최종 상업 송장)
- 함량 = 순도 (24K=0.999, 18K=0.750, 14K=0.585)
- 스폿가격 = 금 시세 기준가, 공임 = 가공 임가공비
- 수출구분: 일반수출 / 추가가공수출 (수금 대상 금액 범위가 다름)

## 3. 화면 구성 (좌측 사이드바 네비)

화이트보드의 ①~⑥ 흐름을 그대로 화면으로 만든다. 사이드바 메뉴 순서:

**대시보드** (홈)
- 상단 KPI 카드: 당월 매출, 미수금 총액, 진행중 주문 수, 오늘 금시세(전일 대비 등락 표시)
- 매출 추이 라인차트(최근 6개월), 수출구분별 매출 도넛차트
- 최근 주문 5건 테이블

**① 주문 접수**
- 주문 목록 테이블(주문번호/거래처/제품/중량/상태/일자)
- 우상단 [+ 신규 주문] [엑셀 업로드] [엑셀 다운로드] 버튼
- 신규 주문 = 모달 폼(거래처 선택, 제품 선택, 중량 입력, 수출구분)
- 상태 뱃지: 접수 / PI발행 / 생산중 / CI발행 / 완료

**② 견적서(PI) 발행**
- 주문 선택 → PI 미리보기(거래처 정보, 품목, 금시세 적용가, 금속가+공임 합계, USD 환산)
- [엑셀 다운로드] [PDF 다운로드] 버튼 → 토스트
- PI 양식은 실제 수출 인보이스처럼: 헤더(회사/거래처), 품목 테이블, 합계, 환율·금시세 명시

**③ 견적 업데이트**
- 선택한 PI의 수량/중량을 +/- 조정
- "실제 로드 중량" 입력 → 금액 자동 재계산(실시간 반영 보여주기)
- 제품 type별 라벨링/명세 표기 토글
- 변경 전/후 금액 비교 표시

**생산발주서 생성** (화이트보드 2-1)
- 견적 확정분 → 생산발주서 미리보기 + [엑셀 다운로드]

**④ 최종 인보이스(CI) 발행**
- PI → CI 전환. **이격 반영**: "PI 시세 → CI 시세, 차액 ±" 강조 표시
- 탭 2개: [고객별 개별 시트] / [Shipping별 종합 시트]
- [엑셀 다운로드] [PDF 다운로드]

**⑤ 매출 관리**
- 매출 실적 테이블(월별/거래처별)
- 매출 분석: 막대/라인 차트(거래처별, 제품별, 월별)
- [엑셀 다운로드]

**⑥ 수금 관리**
- 거래처별 수금/미수금 테이블, 미수금은 빨강 강조
- 수출구분(일반/추가가공) 필터 → 수금 대상 금액 범위 다르게 표시
- [엑셀 다운로드]

**은행 입금내역** (화이트보드 6-1)
- 입금 내역 테이블 + [엑셀 업로드](토스트) → 미수금과 자동 대사된 것처럼 "매칭됨" 뱃지 표시

**마스터 데이터** (서브 메뉴 그룹)
- 제품 spec 관리 / 스폿(금시세) 관리 / 고객 기준정보 / 고객공임 관리 / 금시세 테이블(일자별)
- 각각 테이블 + [추가][수정] 버튼(모달)

## 4. 계산 로직 (데모에 그대로 박아 넣을 예시)

JS에 계산 함수로 넣고, 화면 숫자는 이 함수로 산출해 일관성 있게:

```js
// 24K 금시세(원/g) 기준
function calcMetal(spot24k, purity, weightG) {
  return Math.round(spot24k * purity * weightG);
}
function calcInvoice(spot24k, purity, weightG, laborPerG) {
  const metal = calcMetal(spot24k, purity, weightG);
  const labor = Math.round(laborPerG * weightG);
  return { metal, labor, total: metal + labor };
}
// 워크드 예시: 18K(0.750) 체인 500g, 금시세 151,800, 공임 8,500/g
// 금속가 = 151,800 × 0.750 × 500 = 56,925,000
// 공임   = 8,500 × 500 = 4,250,000
// 합계   = 61,175,000원  →  환율 1,380 → USD 44,329.71
```

## 6. 디자인 방향

- **톤:** 신뢰감 있는 B2B ERP. 데이터 밀도 높은 테이블이 주인공.
- **컬러:** 차분한 슬레이트/네이비 베이스 + **골드 포인트 컬러**(도메인이 금이라 #C8A24B 계열 액센트 한 스푼). 과하지 않게.
- **레이아웃:** 좌측 고정 사이드바 + 상단 페이지 타이틀/액션 버튼 + 본문. KPI 카드는 상단 그리드.
- **테이블:** 줄무늬 없는 깔끔한 행, 상태/금액은 뱃지·색으로 강조(미수금 빨강, 완납 초록).
- **숫자:** 천단위 콤마, 원화는 ₩, 수출액은 USD 병기.
- **마이크로 디테일:** 토스트, 호버 상태, 모달, 등락 화살표(▲▼) — 살아있는 느낌.

## 7. 데모 규칙 (반드시 지킬 것)

1. 모든 데이터는 JS 목업. 서버 호출 없음.
2. 다운로드/업로드 버튼 = 토스트로 처리(실제 파일 X).
3. 화면 간 데이터 연결은 보여주되(주문→PI→CI 흐름), 완벽한 무결성보다 **시각적 설득력** 우선.
4. 빈 화면 금지 — 모든 화면에 그럴듯한 데이터가 차 있어야 함.
5. 회사명은 데모용으로 "(주)○○골드" 같은 가칭 사용, 로고 자리만 표시.

## 8. 빌드 순서 (단계별로 시켜라)

1. 레이아웃 셸: 사이드바 + 라우팅 + 디자인 토큰 + 목업 데이터 객체
2. 대시보드(KPI + 차트 2개 + 최근 주문)
3. 주문 접수 → PI 발행 → 견적 업데이트 (핵심 흐름, 계산 함수 연동)
4. 생산발주서 → CI 발행(이격 반영, 탭 2종)
5. 매출 관리 + 수금 관리 + 은행 입금내역
6. 마스터 데이터 5종
7. 토스트·모달·호버 등 인터랙션 폴리시 + 전체 점검
